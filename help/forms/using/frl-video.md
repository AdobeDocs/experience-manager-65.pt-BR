# Script de vídeo: Configuração do FRL (Feature Restricted Licensing) para o Adobe Acrobat no AEM Forms

| Visual | Título | Script do alto-falante |
|--------|-------|---------------|
| Tela de título com o logotipo do AEM Forms e gráfico FRL | Introdução | Bem-vindo a este tutorial sobre a configuração do Feature Restricted Licensing, ou FRL, para o Adobe Acrobat em seu servidor AEM Forms. Esse processo é essencial à medida que a Adobe transita de licenças perpétuas para o novo modelo de FRL. |
| Linha do tempo mostrando datas de EOL com setas apontando para transição FRL | Fundo | As versões do Adobe Acrobat Classic com licenças permanentes estão atingindo o fim da vida útil, exigindo uma transição para versões mais recentes. O AEM Forms PDF Generator depende do Adobe Acrobat para a funcionalidade de conversão de documentos. O modelo de Licenciamento restrito de recursos fornece a mesma funcionalidade, mas com uma abordagem de licenciamento moderna gerenciada pela Adobe Admin Console. |
| Lista de verificação dos itens necessários com caixas de seleção e ícones | Pré-requisitos | Antes de começarmos, vamos garantir que você tenha tudo de que precisa: primeiro, acesso ao Adobe Admin Console com privilégios de Administrador do sistema. Em segundo lugar, uma conta de usuário com função de Administrador de implantação no Admin Console. Terceiro, privilégios de administrador local no servidor que executa o AEM Forms. Quarto, um sistema operacional Windows de 64 bits em seu servidor AEM Forms. Em quinto lugar, uma conexão estável com a Internet para a ativação de licenças. Familiarize-se com a interface do Adobe Admin Console e entenda a arquitetura de implantação do AEM Forms. |
| Ícone de aviso com gráfico de backup | Observação importante | Lembre-se de fazer backup de qualquer configuração personalizada do Acrobat antes de iniciar, pois será necessário desinstalar as instalações existentes do Acrobat durante esse processo. |
| Diagrama de fluxo mostrando etapas numeradas de 1 a 7 | Visão geral do processo | Esse processo envolve várias etapas principais: preparar um pacote de FRL no Admin Console, conceder permissões de download, desinstalar versões anteriores do Acrobat, instalar o novo Acrobat Pro, implantar o pacote de FRL e verificar a instalação. Vamos analisar cada etapa. |
| Captura de tela da interface do Adobe Admin Console com a guia Pacotes realçada | Etapa 1: Preparação do Admin Console | Primeiro, faça logon no Adobe Admin Console com privilégios de Administrador do sistema. Navegue até a guia Pacotes e selecione &quot;Licença de recurso restrito&quot;. Clique no botão &quot;Introdução&quot; para começar a criar o pacote. |
| Tela de configuração Criar pacote com as configurações realçadas | Configuração do pacote | Configure seu pacote com estas configurações recomendadas: selecione &quot;Offline&quot; para o método de ativação, &quot;Geração de PDF&quot; para direitos e &quot;Windows de 64 bits&quot; para a plataforma. Para seleção de aplicativos, mantenha somente o Arquivo de licença nos aplicativos selecionados. Dê um nome descritivo ao pacote, como &quot;Acrobat FRL AEM Forms&quot;, e crie o pacote. |
| Guia Usuários do Admin Console com caixa de diálogo de permissões | Etapa 2: Conceder Permissões De Download | Em seguida, será necessário criar ou identificar um usuário que baixará o pacote. No Admin Console, navegue até a guia Usuários, selecione seu usuário e atribua a ele a função &quot;Administrador de implantação&quot;. Isso permite que eles baixem o pacote de FRL que você criou. |
| Painel de controle do Windows com aplicativos e recursos mostrando o Acrobat | Etapa 3: Desinstalar o Acrobat anterior | Antes de instalar a nova versão, é necessário remover completamente todas as instalações existentes do Acrobat. Abra o Painel de Controle do Windows, navegue até Aplicativos, localize o Adobe Acrobat e selecione Desinstalar. Para uma remoção completa, considere usar a ferramenta Adobe Acrobat Cleaner, especialmente se você encontrar problemas posteriormente no processo. |
| A página Downloads do Adobe Acrobat DC com a seção do instalador está realçada | Etapa 4: baixar e instalar o Adobe Acrobat Pro | Após a desinstalação, baixe o instalador apropriado do Acrobat Pro na página Downloads do Adobe Acrobat DC. Para compatibilidade com o PDF Generator, recomenda-se o instalador do Windows de 32 bits. |
| Processo de instalação mostrando etapas de extração e configuração | Processo de instalação | Extraia o arquivo zip baixado para uma pasta, localize Setup.exe e execute-o como administrador. Siga as instruções na tela para concluir a instalação. Após a instalação, abra o Acrobat Pro uma vez para fechar as caixas de diálogo de boas-vindas, concluindo a configuração inicial. |
| Guia Pacotes do Admin Console com o botão de download realçado | Etapa 5: baixar o pacote FRL | Agora, faça logon no Adobe Admin Console usando a conta à qual você concedeu permissões de download anteriormente. Navegue até a guia Pacotes, localize o pacote FRL e baixe-o no servidor do AEM Forms. |
| Prompt de Comando como Administrador com comandos sendo digitados | Etapa 6: implantar o pacote | Extraia o pacote baixado para um diretório em seu servidor. Abra o Prompt de Comando como Administrador e navegue até o diretório de extração. |
| Linha de comando que mostra o comando de ativação com a sintaxe realçada | Comando de ativação | Execute o comando de ativação usando esta sintaxe: `adobe-licensing-toolkit.exe -p -i -f ngl-preconditioning-data.json`. Substitua o nome de arquivo JSON pelo nome exato do seu pacote. Os parâmetros de comando são: -p para plataforma, -i para instalar a licença e -f para especificar o caminho do arquivo JSON. Ao obter êxito, você verá a mensagem &quot;Operação concluída com êxito&quot;. |
| Tabela mostrando parâmetros e descrições de comando | Parâmetros de comando | O parâmetro -p especifica a plataforma e detecta automaticamente o sistema operacional. O parâmetro -i instrui a ferramenta a instalar e ativar a licença. E o parâmetro -f especifica o caminho para o arquivo de licença JSON. |
| Interface do administrador do AEM Forms mostrando o serviço do PDF Generator | Etapa 7: testar e verificar | Por fim, teste sua instalação abrindo a interface do administrador do AEM Forms e convertendo um documento simples em PDF usando o serviço PDF Generator. Verifique também a instalação do Acrobat abrindo o Acrobat Pro, navegando até Ajuda → Sobre e confirmando o número da versão e o status da licença. |
| Tela dividida: caixa de diálogo Sobre do Acrobat e conversão bem-sucedida do PDF | Verificação | Verifique se a licença é exibida como ativada no Acrobat e se o PDF Generator pode converter documentos com êxito. |
| Erros comuns e lista de soluções com ícones | Dicas de solução de problemas | Se você encontrar problemas, verifique estes problemas comuns: Verifique se você está executando o Prompt de Comando como administrador, verifique se o nome do arquivo JSON está correto, verifique se há instalações conflitantes e confirme se o servidor tem conectividade adequada com a Internet para ativação. |
| Itens adicionais de solução de problemas com marcas de seleção | Mais Solução de Problemas | Verifique também se há restrições de firewall, pacotes corrompidos ou Adobe IDs conflitantes no servidor. Se tudo falhar, entre em contato com o Suporte da Adobe com mensagens de erro detalhadas. |
| Tela de sucesso com ícone de marca de seleção e informações de contato | Conclusão | Agora você configurou com êxito o Licenciamento restrito de recursos para o Adobe Acrobat no servidor do AEM Forms. Se precisar de assistência adicional, entre em contato com o Suporte da Adobe ou consulte a documentação detalhada disponível no Experience League. |
| Tela de fechamento com logotipo e créditos do Adobe | Obrigado | Obrigado por assistir a este tutorial. Para obter mais informações, consulte a documentação do Experience League. |
